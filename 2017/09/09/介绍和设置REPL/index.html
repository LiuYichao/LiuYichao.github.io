第一部分：介绍和设置REPL

作为一个web开发人员，我每天在工作中都会使用关系型数据库，不过它对于我来说就是一个黑匣子。我又一些问题：

  • 保存数据的格式是怎么样的？（在内存和硬盘中）
  • 它何时从内存移动到磁盘？
  • 为什么每张表只能有一个主键？
  • 回滚事务是如何工作的？
  • 索引的格式是怎样的？
  • 什么时候以及如何进行全表扫描？
  • 预处理语句的保存格式是怎样的？

换句话说，数据库是如何工作的？

为了把事情搞清楚，我从头写一个数据库。他模仿了SQLite， 因为它被设计得比MySQL或PostgreSQL更小、特征更少，于是我更有希望理解它。整个数据库存储在单个文件中！

SQLite

在SQLite的网站中有大量的关于SQLite内部结构的文档，我有一份拷贝：SQLite Database System: Design and Implementation

查询通过一个组件链来检索或修改数据。前端由以下部分组成：

  • tokenizer
  • parser
  • code generator

前端输入的是一个SQL查询，输出的是SQLite虚拟机字节码（本质上是一个编译后的程序，可以操作数据库）

后端由以下部分组成：

  • virtual machine
  • B-tree
  • pager
  • OS interface

虚拟机（virtual machine）使用由前端生成的字节码作为指令。然后，它可以在一个表或多个表或索引上执行操作，每个表或索引都保存在一个叫做B-tree的数据结构中。虚拟机本质上是字节码指令类型上的一个巨大switch语句。

每个B-tree由许多节点组成。每个节点的长度为一页。B-tree可以从磁盘中检索页或者通过发出指令到pager将其保存到磁盘。

pager接收命令来读写页数据。它负责在数据库文件中以适当的偏移量进行读写。它还会降最近访问的页的缓存保存在内存中，并且判定这些页何时该写回磁盘。

OS interface是取决于哪个操作系统编译SQLite而不同的一层。在这份指南中,我不会去支持多平台。

千里之行始于足下，所以，让我们从一些更简单的事开始：REPL
